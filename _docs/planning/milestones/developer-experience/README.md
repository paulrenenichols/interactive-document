# Developer experience — Milestone planning

Created from the [developer-experience exploration](../../explorations/developer-experience/) with the **docs-driven-dev** skill (v1.0.0).

## Summary

This milestone improves **local development** for the project: faster feedback (e.g. live reload in Docker), better tooling (API docs, DB UI, Storybook), and clearer process (rules and skills for the agent).

## Phases

| Phase | Scope |
|-------|--------|
| **01-dev-stack** | Dev Docker with live reload, DB UI (Adminer), optional seed data |
| **02-api-docs** | Live API docs (OpenAPI/Swagger), API log level via env |
| **03-storybook** | Storybook for frontend components (editor, viewer, charts, auth) |
| **04-rules-and-skills** | Cursor rules and skills from _docs processes (phase execution, exploration evaluation, code conventions) |
