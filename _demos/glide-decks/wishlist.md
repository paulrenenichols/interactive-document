**Purpose**
Keep a running list of desired features improvement, each articulated in 1-3 sentences



*Right Click*
-establish conventions for right clicking throughout the authoring views.  The intent of this web app is to allow authoring of right click behavior within interactive visuals, so handling this is complicated

*Select multiple slides*
-Add an MS Office style feature to be able to simultaneously select one slide or multiple slides in the slide or layout thumbnail views.  First selection/focus materializes the slide in the canvas, but the user can also shift or control click to add "selection" state to many slides for bulk options copy/paste/delete


*Deterministic new object placement in slide view*
-Based on whitespace in a current slide, place new content launched from clicking buttons (Add chart, add shape, etc.) centered in the unused whitespace

*allow clipping so elements can partially occlude onto the slide*
-Especially for shapes and occasionaly for charts it's useful in powerpoint to be able to drag the shape partially outside of the viewport
-The current slide deck authoring canvas does not allow movement outside of the slide viewport
-I would like there to be some grace room - maybe 300-600 pixels where you can trag the shape out of visible space, see the bounding box beyond the slide, and still drag it back.  In preview only content within the slide viewport will render
